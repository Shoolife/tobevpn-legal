# Политика конфиденциальности ToBeVPN

**Дата вступления в силу:** 30 апреля 2026 года

[English version](privacy-en.html)

ToBeVPN («мы», «приложение») — это VPN‑сервис для устройств на Android. Доступны клиент для телефона/планшета (`com.tobevpn.app`) и клиент для Android TV (`com.tobevpn.tv.app`). Эта политика распространяется на оба клиента и описывает, какие данные мы собираем, для каких целей и кому передаём.

- **Юрисдикция:** Российская Федерация
- **Контакт по вопросам персональных данных:** [iv.zai4k@gmail.com](mailto:iv.zai4k@gmail.com)

## 1. Принципы

- Мы **не ведём логов вашего интернет‑трафика**: посещённые сайты, доменные имена, содержимое запросов и ответов не записываются и не передаются третьим лицам.
- Мы **не собираем геолокацию**, контакты, фотографии, файлы, историю браузера, сообщения и иные персональные материалы с устройства.
- Мы собираем минимально необходимый набор данных, который позволяет приложению работать: идентифицировать устройство, выдавать подписку и считать использованный трафик.

## 2. Какие данные мы собираем

| Данные | Для чего | Где хранятся |
| --- | --- | --- |
| Идентификатор устройства Android (`Settings.Secure.ANDROID_ID`) | Привязка подписки к конкретному устройству | На сервере бота |
| Имя устройства (`Build.MANUFACTURER` + `Build.MODEL`) | Отображение в списке ваших устройств | На сервере бота |
| Telegram ID | Аутентификация и управление подпиской | На сервере бота и в VPN‑панели |
| Email (опционально, если вы его указали) | Восстановление доступа, уведомления | Локально в зашифрованной БД (SQLCipher), на сервере бота и в VPN‑панели |
| Аппаратный идентификатор (HWID) | Учёт устройств в VPN‑панели | В VPN‑панели |
| Версия ОС, модель устройства, версия приложения | Заголовки HTTP при запросе подписки | В VPN‑панели |
| Объём входящего и исходящего трафика (в байтах) | Расчёт лимитов тарифа | На сервере бота и в VPN‑панели |
| IP‑адрес подключения | Кратковременно — для технической работы VPN | В VPN‑панели и на VPN‑узлах |
| Изображение QR‑кода (телефон сканирует QR‑код ТВ, либо ТВ показывает QR‑код для сканирования телефоном) | Привязка ТВ‑устройства к Telegram‑аккаунту | Локально на устройстве, на серверы не передаётся |

## 3. Чего мы НЕ собираем

- Содержимое VPN‑трафика и DNS‑запросов
- Историю посещённых сайтов и приложений
- Геолокацию устройства
- Контакты, SMS, звонки
- Фотографии, файлы, медиа
- Биометрические данные
- Рекламные идентификаторы

## 3.1. Разрешения Android и для чего они

| Разрешение | Назначение |
| --- | --- |
| `INTERNET`, `ACCESS_NETWORK_STATE` | Установление VPN‑туннеля и отслеживание состояния сети |
| `BIND_VPN_SERVICE` | Использование системного VpnService API для построения VPN‑туннеля |
| `FOREGROUND_SERVICE`, `FOREGROUND_SERVICE_SPECIAL_USE` | Удержание VPN‑соединения активным, пока приложение работает в фоне |
| `POST_NOTIFICATIONS` | Отображение системного индикатора статуса VPN. Уведомления **не используются** для маркетинга, рекламы и сбора аналитики. |

Мобильная версия дополнительно использует встроенный системный сканер кодов **Google Code Scanner** для привязки ТВ‑устройства по QR‑коду. Сканер работает локально, разрешение `CAMERA` приложением не запрашивается, изображения не сохраняются и не передаются на наши серверы.

Версия для Android TV вместо сканирования **локально отрисовывает QR‑код** с помощью открытой библиотеки ZXing. QR содержит ссылку вида `https://t.me/<бот>?start=<auth_token>`, которую пользователь сканирует телефоном для входа через Telegram. Приложение для ТВ так же не запрашивает разрешение `CAMERA`; изображение генерируется и отображается полностью на стороне ТВ.

Глубокая ссылка `tobevpn://auth_callback` (только мобильное приложение) используется для возврата в приложение после успешной авторизации в Telegram.

## 3.2. Декларация VPN‑сервиса

В соответствии с правилами Google Play для приложений, использующих `VpnService` API, мы заявляем:

- VPN‑функциональность является **основной и заявленной** функцией приложения, а не побочной.
- VpnService используется **исключительно** для построения VPN‑туннеля от вашего устройства до наших VPN‑узлов.
- Мы **не модифицируем** содержимое VPN‑трафика, **не внедряем** в него рекламу, аналитику и трекеры.
- Мы **не перенаправляем** трафик пользователя третьим лицам в коммерческих целях.
- Мы **не используем** VpnService как библиотеку в составе других приложений.
- Источник VPN‑протоколов открыт (XRay/Sing‑box). Конфигурация подключения формируется только на основании вашей подписки.

## 4. Передача данных третьим лицам

Данные передаются только следующим обработчикам, без которых сервис работать не может:

- **Telegram** (telegram.org) — для подтверждения авторизации.
- **Серверная инфраструктура ToBeVPN** (бот, VPN‑панель и VPN‑узлы) — для выдачи подписки и работы VPN.
- **Платёжные системы** (Telegram Stars, Cryptomus) — обрабатываются на стороне Telegram‑бота, приложение не получает данные карт или кошельков.
- **Open Exchange Rates** (`open.er-api.com`) — публичный API курсов валют, используется для отображения цены подписки в локальной валюте. Запрос содержит только IP‑адрес устройства; персональные данные не передаются.
- **Maven Central** (`repo1.maven.org`) — публичный репозиторий, используемый как endpoint для теста скорости интернета. Запрос содержит только IP‑адрес устройства; персональные данные не передаются.
- **Google Play** — для распространения приложения и обработки покупок (если применимо).

Мы не продаём ваши данные рекламодателям и не передаём их в маркетинговых целях.

## 5. Хранение и удаление

Сроки хранения по типам данных:

| Данные | Срок |
| --- | --- |
| Telegram ID, email, имя устройства, HWID, привязка устройства | До удаления аккаунта или отвязки устройства |
| Счётчики трафика подписки | На время действия подписки + до 12 месяцев в обезличенном виде для статистики |
| Технические логи серверов (IP, временные метки) | До 90 дней |
| Финансовые и налоговые записи о платежах | До 5 лет (требование законодательства) |

Локальная база данных приложения зашифрована (SQLCipher) и хранится только на вашем устройстве.

**Как удалить аккаунт:** см. отдельную страницу [«Удаление аккаунта»](delete-account.html). Запросы по email обрабатываются в течение 30 дней.

Удаление приложения с устройства автоматически очищает локальные данные, но не разрывает связь устройства с вашим Telegram‑аккаунтом на сервере — для этого воспользуйтесь инструкцией по удалению.

## 6. Безопасность

- Все обращения к серверам идут по HTTPS (TLS 1.2+).
- Локальная БД зашифрована (SQLCipher).
- Серверный API использует токен‑авторизацию.
- В случае инцидента, повлёкшего риск для прав и свобод субъектов данных, мы уведомим затронутых пользователей и компетентный надзорный орган в сроки, предусмотренные применимым законодательством (для GDPR — в течение 72 часов с момента обнаружения, Art. 33–34).
